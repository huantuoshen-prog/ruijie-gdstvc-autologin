## 关于多设备检测与绕过

校园网通常会检测多设备同时上网，但屏蔽方法非常简单：

### 原理说明

校园网通过 **MAC 地址** 来识别设备数量。通过 **NAT (网络地址转换)** 技术，可以让校园网只检测到一个 MAC 地址，所有流量都通过这个地址转发。

### 简易版解决方案（推荐）

使用 OpenWrt 或其衍生版本（如 ImmortalWrt、PandoraBox 等）的路由器，路由器本身已经自带 NAT 配置，默认就会让校园网只识别到一个 MAC 地址。

**操作步骤：**
1. 电脑/手机连接路由器 WiFi 或网线
2. 路由器WAN口连接校园网网口
3. 路由器LAN口/WiFi 连接你的设备
4. 路由器会自动进行 NAT 转换

### 手动检测方法

如果你不确定路由器是否正常工作，可以：

```bash
# 在路由器上执行，查看 WAN 口 MAC
ifconfig eth0 | grep HWaddr

# 或者查看校园网认证页面显示的 MAC 地址
```

如果两者一致，说明 NAT 正常工作 ✅

### 注意事项

- 🚫 不要同时使用路由器 WAN 口和电脑直连校园网
- ✅ 建议所有设备都通过路由器上网

## 安装

### 方法1: 一键配置（推荐）

最简单的安装方式，一键配置并设置自动登录：

```bash
# 下载配置脚本
wget https://raw.githubusercontent.com/17388749803/Ruijie-Auto-Login/main/setup.sh

# 运行配置（需要root权限）
chmod +x setup.sh
sudo sh setup.sh
```

配置过程：
1. 自动检测并安装必要工具
2. 自动下载认证脚本
3. **互动式界面**，输入用户名和密码
4. 自动测试认证
5. **自动配置定时任务**（凌晨5点-7点，每5分钟尝试）

## OpenWrt 路由器定时任务

配合 OpenWrt 路由器实现自动认证：

### 1. 安装 curl

```bash
opkg update
opkg install curl
```

### 2. 设置定时任务

```bash
# 编辑定时任务
crontab -e

# 添加以下行（每5分钟检查一次）
*/5 * * * * /path/to/ruijie_student.sh 你的学号 你的密码 >> /var/log/ruijie.log 2>&1
```

### 3. 查看日志

```bash
tail -f /var/log/ruijie.log
```

## 常见问题

### Q: 提示"需要输入验证码"怎么办？
A: 当前脚本不支持验证码识别，建议在网页端手动认证一次后，再使用脚本。

### Q: 认证失败怎么办？
A: 
1. 检查用户名和密码是否正确
2. 检查网络是否正常
3. 尝试更换检测地址

### Q: 路由器上运行有问题？
A: 确保路由器已安装 curl：`opkg install curl`

### Q: 联通网络无法使用？
A: 请提交 Issue，告诉我们具体情况，我们会尽快跟进。
